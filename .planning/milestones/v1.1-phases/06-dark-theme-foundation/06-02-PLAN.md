  <action>
Create src/components/Slider.tsx as a controlled React component using pointer events (onPointerDown, onPointerMove, onPointerUp) for drag interaction. Do not use native input type="range".

Interface:
```typescript
interface SliderProps {
  min: number
  max: number
  step: number
  value: number
  onChange: (value: number) => void
  width?: number        // defaults to 120
  ariaLabel?: string
}
```

Implementation details:
- Use a `useRef` on the track element to measure its bounding rect for position-to-value calculation
- `setPointerCapture` on pointerdown so drag works if pointer leaves the track
- Value calculation: `ratio = (clientX - rect.left) / rect.width`, `raw = min + ratio * (max - min)`, snap to step: `Math.round(raw / step) * step`, clamp to [min, max]
- Visual structure (all inline styles):
  - Outer container: `position: relative`, `width: {width}px`, `height: 20px`, `cursor: pointer`
  - Track: `position: absolute`, `top: 50%`, `transform: translateY(-50%)`, `width: 100%`, `height: 4px`, `background: #2e2e4a`, `borderRadius: 2px`
  - Thumb: `position: absolute`, `top: 50%`, `transform: translateY(-50%)`, left positioned as `${((value - min) / (max - min)) * 100}%` with `translateX(-50%)` applied via a combined transform — use a separate `thumbLeft` calculation: `((value - min) / (max - min)) * width - 8` pixels from left
  - Thumb size: 16px x 16px, `borderRadius: '50%'`, `background: #4488ff`, no border
  - No fill bar from the left — track is uniform color
- Accessibility: `role="slider"`, `aria-valuemin`, `aria-valuemax`, `aria-valuenow`, `aria-label` on the outer div

Write tests in src/components/__tests__/Slider.test.tsx using vitest + @testing-library/react. Test: value-to-position mapping, onChange called with snapped value, clamping at boundaries.
