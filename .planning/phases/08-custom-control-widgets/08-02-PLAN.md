  <files>src/hooks/useAnimation.ts</files>
  <action>
Add pause/resume/step capability to useAnimation. The key design: use a `isPausedRef` (ref, not state) to gate the rAF loop without stopping it, and a `stepRef` flag to advance exactly one frame when stepping.

Implementation approach:

1. Add `const isPausedRef = useRef(false)` and `const [isPaused, setIsPaused] = useState(false)`.

2. In the `frame()` closure, at the very top of the function body, add:
```ts
if (isPausedRef.current && !stepRef.current) {
  rafHandleRef.current = requestAnimationFrame(frame)
  return
}
stepRef.current = false
```
This keeps the rAF loop alive while paused, but skips all work (no cursor advance, no layer updates) unless `stepRef.current` is true.

3. Add `const stepRef = useRef(false)`.

4. `pauseAnimation`: sets `isPausedRef.current = true`, calls `setIsPaused(true)`.

5. `resumeAnimation`: sets `isPausedRef.current = false`, calls `setIsPaused(false)`.

6. `stepAnimation`: sets `stepRef.current = true` (the next frame will execute exactly once then re-enter paused state).

7. In `cancelAnimation` and `startAnimation`: reset `isPausedRef.current = false`, `stepRef.current = false`, `setIsPaused(false)` — so starting a new animation always starts unpaused.

8. Update the return type to include: `isPaused: boolean`, `pauseAnimation: () => void`, `resumeAnimation: () => void`, `stepAnimation: () => void`.

Note: `isPausedRef` and `stepRef` must be declared OUTSIDE of `startAnimation`'s `useCallback` (at hook level) so they persist across re-renders. The `frame` closure inside `startAnimation` captures them by ref — this works because refs are stable objects.
