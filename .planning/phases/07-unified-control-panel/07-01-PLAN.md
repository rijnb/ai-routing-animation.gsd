Create `src/components/ControlPanel.tsx` as a new component. It accepts these props:

```tsx
interface ControlPanelProps {
  // File loading
  onFile: (file: File) => void
  isLoading: boolean
  // Routing controls
  geojson: unknown | null          // non-null = file loaded = show routing controls
  mode: RoutingMode
  onModeChange: (mode: RoutingMode) => void
  speed: number
  onSpeedChange: (v: number) => void
  route: unknown | null            // non-null = route active = show SpeedPanel
  onReload: () => void             // called when user clicks "Load new file"
}
```

**Panel shell** (position and visual style from 07-CONTEXT.md decisions):
- `position: 'fixed'`, `bottom: 16`, `right: 16`, `zIndex: 500`
- `width: 300` (fixed, does not change between states)
- `background: '#1a1a2e'`
- `border: '1px solid rgba(255,255,255,0.1)'`
- `boxShadow: '0 4px 16px rgba(0,0,0,0.4)'`
- `borderRadius: 4`
- `padding: '10px 12px'`
- `overflow: 'hidden'`

**State: drop zone visible** (`geojson === null && !isLoading`):
- Implement the drop zone UI directly inside ControlPanel (not via the DropZone component) to avoid DropZone's own `position: absolute` wrapper conflicting with panel flow.
- Copy the drag-over logic (useState for dragging, onDragOver/onDragLeave/onDrop handlers) and the same visual styles from DropZone.tsx: dashed border, drag-over highlight (`rgba(68,136,255,0.12)` background), `#aabbff` text, 📂 icon, "or click to browse" subtext, hidden file input, and the two bundled map buttons ("Load Leiden", "Load Amsterdam").
- The bundled map button handler fetches `/maps/{filename}` and calls `onFile` with the resulting File — same logic as in DropZone.tsx.
- The original `DropZone.tsx` file remains on disk unchanged (it is no longer rendered by App.tsx but can stay for reference).

**State: routing controls visible** (`geojson !== null`):
- Render `<ModeSelector mode={mode} onModeChange={onModeChange} />` spanning full panel width.
- Below it: a row with `<SpeedPanel speed={speed} onSpeedChange={onSpeedChange} visible={route !== null} />` on the left.
- Below the controls: a small "Load new file" button — `color: 'rgba(170,187,255,0.5)'`, `fontSize: 12`, `background: 'none'`, `border: 'none'`, `cursor: 'pointer'`, `padding: '4px 0 0 0'`, `display: 'block'` — clicking calls `onReload`.

**State: loading** (`isLoading`): render a simple loading text inside the panel: `<div style={{ color: 'rgba(170,187,255,0.5)', fontSize: 13, padding: '8px 0' }}>Loading...</div>`.

**CSS transitions between states**: Wrap drop zone content and routing controls content each in a `div` with `transition: 'opacity 0.25s ease'`. Show the active state at `opacity: 1` and hide the inactive at `opacity: 0`. For the panel height, apply `transition: 'max-height 0.3s ease'` on the panel shell: `maxHeight: 260` when showing drop zone, `maxHeight: 180` when showing routing controls.

Import `RoutingMode` from `'../lib/router'`. Import `ModeSelector` from `'./ModeSelector'`. Import `SpeedPanel` from `'./SpeedPanel'`. Add `useRef, useState` from `'react'`.

Follow existing inline `style={{...}}` pattern throughout — no CSS modules or Tailwind.
