    Rewrite `src/workers/osmWorker.ts` to add typed message dispatch and routing support.

    Worker-side state (module-level, between messages):
    ```typescript
    let osmGraph: OsmGraph | null = null
    let adjacency: AdjacencyList | null = null
    ```
    Note: componentMap is sent to main thread at load time (it's small — just a string→string map). adjacency stays worker-side (Pitfall 6: too large to transfer).

    Message handler with type discriminator:
    ```typescript
    self.onmessage = (event: MessageEvent) => {
      const { type, ...payload } = event.data
      if (type === 'load') { ... }
      if (type === 'route') { ... }
    }
    ```

    'load' handler (migrated from existing logic):
    - Accept payload.buffer: ArrayBuffer (not raw event.data)
    - Decompress, parse, build GeoJSON (same as before)
    - Also call buildAdjacency(graph) → store adjacency worker-side, get componentMap
    - Post { type: 'done', geojson, componentMap } — adds componentMap to existing done message
    - Store osmGraph and adjacency in module-level vars for subsequent route messages

    'route' handler (new):
    - Accept payload: { source: [number,number], destination: [number,number], mode: RoutingMode }
    - If !adjacency or !osmGraph: post { type: 'route-error', message: 'No graph loaded' }
    - Snap source and dest using snapToNearestSegment(payload.source, osmGraph, payload.mode, 200)
      — NOTE: snap runs in the worker here too (needs OsmGraph which is worker-side)
      — If snap returns null for either point: post { type: 'route-error', message: 'No road within 200m' }
    - Inject virtual nodes into a SHALLOW COPY of adjacency (never mutate shared state — Pitfall 3):
      VIRTUAL_START = '__vs__', VIRTUAL_END = '__ve__'
      Add virtual start edges to segmentNodeA and segmentNodeB of source snap
      Add virtual end edges from segmentNodeA and segmentNodeB of dest snap back-edges
      Include back-edges from the real nodes to the virtual nodes
    - Run aStar(virtualAdjacency, '__vs__', '__ve__', nodes, payload.mode)
    - Prepend snappedSource and append snappedDest coords to result.path
    - Delete virtual node keys from virtualAdjacency after A* completes
    - Post { type: 'route-done', path: result.path, searchHistory: result.searchHistory, distance: result.distance, found: result.found }

    IMPORTANT: useOsmLoader.ts currently sends raw ArrayBuffer to the worker without a type field. The worker must handle BOTH the old format (backwards compatibility during transition) AND the new typed format. Add: if no event.data.type, treat it as a 'load' message with buffer = event.data.

    Update progress messages to include new "Building routing graph..." stage at ~85% during load.

  <action>
    Create `src/hooks/useRouter.ts`:

    The hook DOES NOT own the worker — the worker is owned by useOsmLoader. Instead, useRouter receives the worker ref as a parameter alongside graph and componentMap:

    ```typescript
    export function useRouter(
      workerRef: React.RefObject<Worker | null>,
      graph: OsmGraph | null,
      componentMap: ComponentMap | null,
    ): RouterState
    ```

    Internal state:
    - mode: RoutingMode (default 'car')
    - clickCount: number (0, 1, 2) — cyclic: 0=next is source, 1=next is dest, 2=next replaces source
    - sourceSnap: SnapResult | null
    - destSnap: SnapResult | null
    - route: RouteResult | null
    - routeError: string | null

    handleMapClick(lngLat: [number, number]):
    - If !graph: return (no graph loaded yet)
    - snapResult = snapToNearestSegment(lngLat, graph, mode, 200)
    - If !snapResult: setRouteError('No road within 200m'); return
    - clickCount % 2 === 0 → setSourceSnap(snapResult); increment click counter
    - clickCount % 2 === 1 → setDestSnap(snapResult); increment click counter
    - After setting either snap: if both sourceSnap and destSnap are non-null, trigger route

    triggerRoute() (internal):
    - If !componentMap || !sourceSnap || !destSnap: return
    - Check connectivity: componentMap[sourceSnap.segmentNodeA] !== componentMap[destSnap.segmentNodeA]
      (use segmentNodeA as the representative; if different components, warn and return)
    - If disconnected: setRouteError('No route — points are on disconnected road segments'); return
    - Post to worker: workerRef.current?.postMessage({ type: 'route', source: sourceSnap.snappedPoint, destination: destSnap.snappedPoint, mode })
    - Worker response handled via useEffect subscribing to worker messages

    Worker message subscription (useEffect on workerRef):
    - Listen for route-done: setRoute(data), setRouteError(null)
    - Listen for route-error: setRouteError(data.message)

    Mode change (setMode):
    - Update mode state
    - If both sourceSnap and destSnap exist: automatically trigger re-route (per locked decision: mode change triggers immediate re-route)

    resetRouting():
    - Reset all state to initial values (called by App on new OSM file load)

    IMPORTANT: useOsmLoader also uses the same worker. useRouter must NOT create a second worker — it piggybacks on the same worker via the passed workerRef. To avoid message handler conflicts, useOsmLoader's handler should pass through unrecognized message types; useRouter listens separately. Pattern: both add their own onmessage listeners via worker.addEventListener('message', handler) rather than replacing worker.onmessage.

    NOTE: This requires updating useOsmLoader.ts to use addEventListener instead of direct assignment, so both hooks can coexist. Make that change in useOsmLoader.ts.
